# Система загрузки фотографий - Инструкция

## Что было реализовано

### Backend изменения:
1. **Multer конфигурация** (`server/middleware/multer.js`) - обработка multipart загрузок
2. **Миграция БД** - добавлены колонки:
   - `secondary_image` - вторичное изображение проекта
   - `gallery_images` - строка с путями к фото галереи (через запятую)
3. **API endpoints** обновлены:
   - `POST /api/projects` - теперь принимает файлы через multipart/form-data
   - `PUT /api/projects/:id` - обновление проекта с новыми файлами
   - `GET /uploads/*` - статические файлы доступны через этот путь

### Frontend изменения:
1. **AdminPanel.vue**:
   - Новые file input поля для загрузки фотографий
   - Поддержка загрузки через FormData вместо JSON
   - Отображение загруженных файлов с полными URL
   - Поддержка множественной загрузки фото для галереи

2. **WorkDetail.vue**:
   - Автоматическое преобразование относительных путей в полные URL
   - Отображение фото галереи в миниатюрах (thumbnails)

## Как использовать

### Запуск системы:

```bash
# 1. Установить зависимости backend
cd server
npm install

# 2. Запустить миграцию (если еще не запускали)
npm run migrate-db

# 3. Запустить backend
npm run dev

# 4. В другом терминале запустить frontend dev сервер
npm run dev
```

### Загрузка проектов через админку:

1. Перейти на http://localhost:5173/login
2. Войти с учетными данными:
   - Email: `admin@kotinstudio.ru`
   - Password: `admin123`
3. Перейти на вкладку "Портфолио"
4. Нажать "+ Новый проект"
5. Заполнить форму:
   - **Название проекта** (обязательно)
   - **Подзаголовок**
   - **Главное изображение** (upload файл)
   - **Вторичное изображение** (опционально)
   - **Фотографии для галереи** (опционально, можно выбрать несколько)
   - **Описание проекта**
   - **ID предыдущего / следующего проекта** (для навигации)
6. Нажать "Добавить проект"

### Структура загруженных файлов:

Все загруженные файлы сохраняются в папке `server/uploads/` с именами:
- `[timestamp]-[randomId].[расширение]`
- Пример: `1723456789-123456789.jpg`

В БД хранятся пути:
- `main_image`: `/uploads/[filename]`
- `secondary_image`: `/uploads/[filename]`
- `gallery_images`: `/uploads/[file1],/uploads/[file2],/uploads/[file3]`

### Отображение на фронте:

- Главное изображение отображается на страницах проектов
- Вторичное изображение доступно через `project.secondary_image`
- Фото галереи отображаются как миниатюры на странице WorkDetail

## Структура БД

```sql
ALTER TABLE projects ADD COLUMN secondary_image VARCHAR(255);
ALTER TABLE projects ADD COLUMN gallery_images LONGTEXT;
```

Таблица projects теперь содержит:
- `id` - ID проекта
- `name` - название
- `subtitle` - подзаголовок
- `description` - описание
- `main_image` - путь к главному изображению
- `secondary_image` - путь ко вторичному изображению (опционально)
- `gallery_images` - строка путей к фото галереи через запятую
- `prev_id` / `next_id` - для навигации между проектами
- `created_at` / `updated_at` - временные метки

## Примечания

- Максимальный размер файла: **50MB**
- Поддерживаемые форматы: все изображения (jpg, png, webp, gif и т.д.)
- Одновременная загрузка до 20 фото в галерею
- Все пути к файлам сохраняются как относительные в БД
- Frontend автоматически конвертирует их в полные URL `http://localhost:5000/uploads/[filename]`
